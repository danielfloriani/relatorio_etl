\chapter{Glossário de termos técnicos}
Este apêndice tem por objetivo apresentar as definições dos principais termos técnicos, conceitos e jargões da área de Engenharia de Dados utilizados ao longo do texto.

\begin{itemize}
    \item \textbf{API (\textit{Application Programming Interface}):} Conjunto de definições e protocolos que permite a comunicação e a integração entre diferentes sistemas ou aplicativos de \textit{software}.
    \item \textbf{Arquitetura \textit{Medallion}:} Padrão de projeto de organização de dados que divide os arquivos em camadas lógicas (comumente chamadas de \textit{Bronze, Silver} e \textit{Gold}), com o objetivo de melhorar progressivamente a estrutura e a qualidade das informações a cada etapa do processamento \cite{gujjala_medallion}.
    \item \textbf{\textit{Business Intelligence} (BI):} Conjunto de teorias, metodologias e tecnologias (como o Power BI ou Tableau) que transformam dados brutos em informações significativas e úteis para a tomada de decisão estratégica do negócio.
    \item \textbf{Camada \textit{Bronze} (L0):} Camada inicial de armazenamento na Arquitetura \textit{Medallion} onde os dados brutos (\textit{raw data}) são mantidos em seu estado original, exatamente como foram extraídos das fontes, sem aplicação de filtros ou transformações estruturais \cite{gujjala_medallion}.
    \item \textbf{Camada de Apresentação (\textit{View}):} Camada lógica operando sobre a camada \textit{Gold}, responsável por filtrar, deduplicar e expor os dados finais de maneira versátil para os analistas e ferramentas de visualização \cite{reis22}.
    \item \textbf{Camada \textit{Gold} (L2):} Camada final onde os dados, já limpos e validados, são agregados e transformados com base em regras de negócio específicas, ficando prontos e otimizados para o consumo analítico e a geração de relatórios \cite{gujjala_medallion}.
    \item \textbf{Camada \textit{Raw} (Crua):} Zona de pouso inicial (\textit{landing zone}) de um \textit{pipeline} de dados, responsável por receber e armazenar os arquivos externos exatamente em seu formato original, antes de qualquer processamento.
    \item \textbf{Camada \textit{Silver} (L1):} Camada intermediária na qual os dados provenientes da camada \textit{Bronze} passam por processos de limpeza, padronização, deduplicação e validação de qualidade contra contratos predefinidos \cite{gujjala_medallion}.
    \item \textbf{Contrato de Dados (\textit{Data Contract}):} Acordo técnico e formal que define a estrutura, o formato, os padrões de qualidade e o comportamento esperado dos dados na transição entre sistemas ou camadas, garantindo a imposição de esquemas (\textit{schema enforcement}) para a confiabilidade do sistema \cite{armbrust20}.
    \item \textbf{\textit{Data Warehouse}:} Repositório centralizado e estruturado de dados projetado para suportar análises corporativas históricas e relatórios estratégicos \cite{kimball13}.
    \item \textbf{\textit{DataFrame}:} Estrutura de dados tabular bidimensional e heterogênea, amplamente utilizada na linguagem Python (como na biblioteca Pandas), onde os dados são alinhados em linhas e colunas.
    \item \textbf{ETL (\textit{Extract, Transform, Load}):} Processo tradicional de integração de dados que envolve a extração de informações de diversas fontes, sua transformação para um formato adequado e, por fim, o carregamento em um repositório centralizado, prática otimizada pelas modernas arquiteturas de \textit{pipelines} \cite{gujjala_medallion}.
    \item \textbf{\textit{Fail-fast} (Falhar Rapidamente):} Princípio de engenharia de \textit{software} no qual o sistema é projetado para interromper a sua execução imediatamente ao encontrar uma quebra de contrato estrutural, evitando a propagação de falhas em cascata para camadas analíticas.
    \item \textbf{\textit{Fan-out}:} Fenômeno indesejado em operações de cruzamento ou transformação de bases de dados (como despivoteamentos e união de matrizes) no qual o número de instâncias resultantes multiplica-se de forma incorreta \cite{reis22}.
    \item \textbf{Granularidade:} Nível de detalhe, divisão ou resumo dos dados em uma estrutura. Ao longo de um \textit{pipeline}, a granularidade pode transitar de estritamente unitária (um relatório por arquivo) para abrangente e consolidada (múltiplas fontes em um único repositório).
    \item \textbf{Ingestão de Dados:} Processo de coleta e importação de dados de diversas origens (como APIs, arquivos em lote ou bancos de dados) para um ambiente de armazenamento ou processamento primário (como a Camada \textit{Bronze}) \cite{polisetty23}.
    \item \textbf{\textit{Logs}:} Registros textuais e estruturados gerados em tempo real pela execução de um \textit{software}, utilizados para auditar comportamentos, monitorar o desempenho e rastrear os motivos exatos de falhas técnicas.
    \item \textbf{\textit{Melt} (Despivoteamento):} Operação de transformação lógica que converte variáveis dispostas horizontalmente em linhas, alterando a estrutura da matriz para um formato longo (\textit{long format}), padrão ideal para motores de \textit{Business Intelligence}.
    \item \textbf{\textit{Outlier}:} Ponto ou instância de dado que difere expressivamente de outras observações em uma mesma amostra ou série histórica, podendo indicar tanto uma anomalia matemática de extração quanto uma mudança real de mercado na origem.
    \item \textbf{\textit{Parser}:} Algoritmo ou rotina desenvolvida para analisar uma estrutura de dados de entrada (como um arquivo de texto, CSV ou PDF complexo) e extrair os parâmetros e informações relevantes de forma legível para o processamento subsequente.
    \item \textbf{\textit{Pipeline} de Dados:} Conjunto automatizado de processos e fluxos que realiza a extração, o processamento, a validação e o carregamento de dados de um sistema de origem para um ambiente de destino, garantindo o fluxo contínuo e a qualidade das informações \cite{polisetty23}.
    \item \textbf{Qualidade de Dados (\textit{Data Quality}):} Medida da condição dos dados baseada em fatores como precisão, completude, consistência e confiabilidade, assegurada através de processos de validação e limpeza implementados durante o fluxo de dados \cite{polisetty23}.
    \item \textbf{Reconciliação de Dados (Validação Horizontal e Vertical):} Técnicas de auditoria matemática utilizadas para garantir que a soma de partes de um registro corresponda de forma exata aos totais informados pela fonte original, mitigando erros de extração \cite{polisetty23}.
    \item \textbf{\textit{Schema} (Esquema):} Estrutura lógica que atua como a ferramenta de validação de um contrato de dados, especificando os nomes das colunas, os tipos de dados exigidos e as restrições de obrigatoriedade, sendo essencial para lidar com a evolução estrutural em sistemas de \textit{Big Data} \cite{liao20}.
    \item \textbf{\textit{Webhook}:} Mecanismo de comunicação entre sistemas que permite que uma aplicação (como um \textit{bot} do Telegram) envie informações ou comandos em tempo real para outra (como o GitHub) assim que um evento específico ocorre.
\end{itemize}
